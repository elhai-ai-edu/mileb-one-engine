# Knowledge Container Lock

## Purpose

This document locks the project-wide container map for the current stabilization pass.

## Final Container Taxonomy

All major materials must belong to one of these containers:

1. Architecture Rules And Contracts
2. Shared Capability Libraries
3. Product Orchestration Protocols
4. Bot And Feature Knowledge Packs
5. Content And Pedagogical Assets

## Overlap Rule

If a source contains material from more than one container, the source may remain mixed as history, but extracted canonical outputs must be split by container.

## Residual Ambiguities Resolved

- Bot Architect questionnaire -> Product Orchestration Protocols
- skill diagnosis research -> Shared Capability Libraries unless it becomes a cross-system contract
- transformation recipes -> Shared Capability Libraries plus Product Protocols where flow logic is involved
- continuity and evidence object boundaries -> Architecture Rules And Contracts plus companion shared object libraries where reusable structures are needed

## Final Statement

The container map is locked for current work.

New material should be classified into this taxonomy before extraction or promotion.
