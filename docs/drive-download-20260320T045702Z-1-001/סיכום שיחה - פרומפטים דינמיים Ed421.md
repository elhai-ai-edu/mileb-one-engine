סיכום שיחה – פיתוח פרומפטים דינמיים בפרויקט Ed421

# 1. מטרת השיחה

הגדרת תהליך לשליחת פרומפט מותאם אישית ל־GPT API עבור כל משתמש, בהתבסס על נתונים אישיים, רגשיים ותפקודיים.

# 2. יצירת פרומפט דינמי

נבנתה תבנית פרומפט הכוללת שדות משתנים (כגון {{שלב}}, {{רגש}}, {{מיומנות}}), שמתמלאים אוטומטית מתוך Make לפי הנתונים מהשאלון.

# 3. קובץ תגובות לפי מיומנות ורגש

נבנה קובץ CSV עם תגובות תומכות רגשית לפי שלב, מיומנות ורגש. כל תגובה מותאמת להקשר של המשתמש.

# 4. העלאה ל־Google Sheets

הקובץ הוכן בפורמט CSV, וניתנו הנחיות להעלאה ל־Google Sheets לצורך שליפה מתוך Make.

# 5. שימוש ב־Make

בחרת לעבוד עם GPT API ומנוי בסיסי של Make. תרחיש הכולל: 
1. שליפה מהשאלון 
2. חיפוש תגובה תואמת 
3. הרכבת פרומפט 
4. שליחה ל־GPT 
5. שמירת תגובה חזרה למערכת.

# 6. שאלות מרכזיות שעלו

- איך לפרק מרכיבי פרומפט לרמות שונות של דינמיות
- איפה לנהל טקסטים מוכנים – בפרומפט או בקובץ נלווה
- כיצד לשלוח בקשה ל־GPT דרך Make ולשלוט בתוכן
- הבדל בין Prompt Template לבין Prompt שלם

# 7. פעולות שבוצעו

- נבנה מסמך Word עם תבנית פרומפט
- נוצר קובץ תגובות מותאם (CSV)
- הקובץ נשלח והועלה ל־Google Sheets
- הוגדרה המשימה הבאה: בניית שלב Make עם שליפת תגובה והתממשקות ל־GPT API

# 8. המשך מומלץ

- לבנות טבלת תגובות מלאה
- להגדיר תרחיש מלא ב־Make
- לשלב Notion להצגת התגובה למשתמש
