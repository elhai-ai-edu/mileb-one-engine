# Instructor Live Flow — Real-Time Classroom Decision System

## 1. Purpose

This document is a **live decision sheet** for the instructor during class.

It answers one question:

> מה לעשות עכשיו — בזמן אמת?

---

# 🧭 CORE PRINCIPLE

```text
Observe → Diagnose → Act (one action only) → Return to speaking
```

---

# ⏱️ LIVE FLOW (Minute-by-Minute Thinking)

## 🔹 START OF ACTIVITY

Ask:
"האם כולם מדברים?"

### If YES:
→ Continue

### If NO:
→ Activate:

"כולם — משפט אחד: מה הבעיה שלכם"

---

## 🔹 DURING BOT PREPARATION

Observe:
- האם התשובות קצרות?
- האם יש שתיקות?

### If short answers:
Action:
"תוסיף דוגמה אחת"

### If silence:
Action:
"רק משפט אחד"

---

## 🔹 DURING ROLEPLAY

Ask:
"האם יש מאמץ קוגניטיבי?"

### If too easy:
→ Increase challenge

"למה שאני אשלם על זה?"

### If too hard:
→ Reduce load

"תסביר במשפט אחד"

---

## 🔹 DURING PRESENTATION

Ask:
"האם יש מבנה ברור?"

### If no structure:
→ Intervention:

"עצירה — תגיד רק את הבעיה"

---

## 🔹 DURING AUDIENCE QUESTIONS

Ask:
"האם הקהל פעיל?"

### If NO:
→ Assign question:

"קבוצה אחת — שאלת לקוח"

---

## 🔹 DURING RESPONSE TO QUESTIONS

Ask:
"האם הסטודנט עונה או נתקע?"

### If stuck:
→ Action:

"תתחיל: הפתרון שלי עוזר כי..."

---

## 🔹 FEEDBACK MOMENT

Rule:
- 2 points only

Structure:

"משהו טוב → משהו לשיפור"

---

## 🔹 SECOND ROUND

Ask:
"מה לשפר עכשיו?"

→ One instruction only

"תתמקד בדוגמה"

---

# 🧠 DECISION MATRIX

| מצב | זיהוי | פעולה |
|-----|------|------|
| שקט | אין דיבור | משפט אחד |
| קצר | תשובה קצרה | דוגמה אחת |
| תקוע | היסוס | משפט פתיחה |
| קל מדי | אין מאמץ | שאלת ביקורת |
| קשה מדי | בלבול | פירוק למשפט אחד |
| קהל פסיבי | אין שאלות | הקצאת שאלה |

---

# 🔥 GOLD RULES

1. פעולה אחת בלבד
2. לא להסביר — להפעיל
3. להחזיר תמיד לדיבור
4. לא לתקן תוך כדי דיבור

---

# 🎯 FINAL GOAL

Every student speaks multiple times.

Not perfectly — but actively.
