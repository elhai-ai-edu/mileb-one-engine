# Shared Library Ownership Lock

## Purpose

This document locks ownership statements for the active shared libraries.

## Skills Library

- Owns: skill meaning, classification, diagnosis, progression, evidence semantics
- Consumes: course context, population overlays, product references
- Must not redefine: product workflow, credential thresholds, transformation logic

## Transformation Library

- Owns: transformation templates, levels, families, transformed unit logic, gap checks
- Consumes: skills as targets or evidence anchors
- Must not redefine: skill meaning, credential meaning, architecture law

## Credential Logic Library

- Owns: credential units, threshold logic, stackability, recognition states, review modes
- Consumes: skill meaning and evidence semantics from the skills layer
- Must not redefine: pedagogical evidence meaning, product runtime sequence

## Evidence And Continuity Library

- Owns: evidence transport, handoff objects, validation-linked continuity state, stage transition traceability
- Consumes: shared references from skills, credentials, and product protocols
- Must not redefine: what evidence means pedagogically, what credentials mean, or what protocol stages mean

## Product Protocol Registry

- Owns: operating sequence, stage order, checks, intake/output structure
- Consumes: skills, transformation, credentials, evidence/continuity objects
- Must not redefine: architecture law, shared evidence semantics, credential thresholds

## Final Statement

Ownership is now locked at the shared-library layer for the current stabilization pass.

If a concept appears in more than one place, these ownership statements decide which layer is authoritative.
