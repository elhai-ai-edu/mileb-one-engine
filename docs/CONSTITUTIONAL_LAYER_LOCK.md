# Constitutional Layer Lock

## Purpose

This document locks the constitutional layer as final architecture law for the current stabilization phase.

Its job is to stop architecture drift.

## Final Constitutional Scope

The following remain architecture law:

- One-Engine Principle
- immutable kernel rules
- mandatory system-wide governance boundaries
- branch contracts and their non-negotiable relationship
- Lesson Space as the runtime meeting layer
- evidence and continuity as cross-system contract territory
- platform ownership rules

## Explicitly Outside Constitutional Scope

The following do not belong to architecture law:

- skill catalogs and rich diagnosis libraries
- transformation pattern families
- credential recipes and local award rules
- product-specific operating flows
- bot-local scaffolds and domain packs

## Locked Decision

If a concept does not define a platform-wide constraint, shared runtime contract, or non-optional governance rule, it should not be promoted into the constitutional layer.

## Residual Drift Candidates Downgraded

These should remain below architecture unless future interoperability needs force promotion:

- active-learning pattern catalogs
- pedagogical DNA registry content beyond cross-system enforcement implications
- population adaptation content beyond routing contracts
- local credential recipes

## Final Statement

The constitutional layer is now treated as small, strict, and closed for this stabilization pass.
