# Secondary Library Cycle 2 Plan

## Purpose

This document opens the second stabilization cycle.

The first cycle closed the active core layers.

The second cycle focuses on secondary but strategically important libraries that still remain under-formalized.

## Cycle 2 Scope

The current cycle covers three libraries:

1. Bot Archetype And DNA Registry
2. Active Learning And Skills-Oriented Design Pattern Library
3. Population Routing And Adaptation Library

## Why These Three Come Next

They are no longer core ambiguity blockers.

But they still carry reusable knowledge that should not stay fragmented inside raw documents, audits, or bot-local packs.

## Work Order

### Task A: Bot Archetype And DNA Registry

- Goal: stabilize the pedagogical DNA layer below architecture and above product-local bot designs
- Main sources: `docs/STRATEGIC_MAP_V3.md`, `docs/KNOWLEDGE_AUDIT_V3.md`, `docs/BOT_ARCHITECT_SP.md`
- Done when: DNA types, archetype families, typical phases, and enforcement tendencies are described in one governed registry
- Status: completed in draft form

### Task B: Active Learning And Skills-Oriented Design Pattern Library

- Goal: extract reusable pedagogical patterns that currently sit scattered across theory docs, redesign notes, and bot-local methods
- Main sources: theoretical continuum docs, ED421-like materials, scaffolding / inquiry / reflection sources
- Done when: core pattern families, selection logic, and pattern-to-evidence directions are available as a shared library
- Status: completed in draft form

### Task C: Population Routing And Adaptation Library

- Goal: normalize population-sensitive routing logic without pushing local pedagogy into architecture law
- Main sources: `docs/opal-populations.md`, `docs/opal-library-map.md`, literacy and adaptation materials
- Done when: routing dimensions, adaptation overlays, and population-specific support logic are described as reusable structures
- Status: completed in draft form

## Cycle 2 Completion Status

The operational checklist for this cycle has been completed at draft-library level:

- registry/library drafts exist for all three targets
- companion contracts now exist for all three targets
- inter-library alignment has been updated
- product protocol dependencies have been updated
- migration control and consistency pass documents now exist for Cycle 2

## Completion Rule

This cycle should produce at least:

- one registry/library document per target library
- an ownership statement inside each library
- enough extracted structure to support future contracts or protocol dependencies

## Final Statement

Cycle 2 is not about repairing the core.

It is about preventing the next wave of drift by formalizing the most reusable secondary knowledge domains.
