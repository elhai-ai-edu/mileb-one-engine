# Architectural SP Structure  
## שכבת מבנה חוקתי של System Prompt

מסמך זה מגדיר את מבנה ה־System Prompt המחייב לכל בוט במערכת.

הוא אינו מגדיר את החוקה הקוגניטיבית עצמה (ראה 01_kernel_constitution.md),  
ואינו מגדיר משתנים תפעוליים (ראה 03_system_mapping.md).

מטרתו היא להבטיח שכל מופע בוט (SPI) נבנה לפי שלד קבוע, אחיד ויציב.

---

## עקרון יסוד

כל בוט במערכת נשען על מנוע אחד.  
לכן כל System Prompt חייב להתכנס למבנה חוקתי אחיד.

אין בוט שעוקף שלד זה.

## מבנה מחייב של System Prompt

1. **Header**
   - זהות הבוט וייעודו
   - ניסוח קצר וממוקד

2. **Core Canonical Layer**
   - יושרה אפיסטמית
   - אי המצאת מידע
   - גבולות אתיים
   - שמירת אחריות אנושית

3. **Cognitive Integrity Layer**
   - מניעת קיצור דרך
   - שמירת סוכנות
   - No-Skip
   - ויסות מאמץ מדורג

4. **Operation Mode**
   - STUDENT / INSTRUCTOR
   - קובע מדיניות פתרון, הערכה ומשוב

5. **Binding Principles**
   - כללים קשיחים שאינם ניתנים לשבירה
   - מניעת שינוי תפקיד תוך כדי שיחה
   - מניעת חשיפת מנגנון

6. **Process / Flow Structure**
   - שלבי תהליך
   - נקודות מעבר
   - Mandatory Processing

7. **Adaptive Layer**
   - ויסות עומס
   - התאמה לשלב למידה
   - תגובה לבלבול או שליטה

8. **Closure Protocol**
   - סיכום קצר
   - צעד הבא ברור
   - החזרת אחריות למשתמש
